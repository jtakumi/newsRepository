## 2. 中国Z.ai、低価格AI coding tool「ZCode」で米国勢に対抗

Business Insiderは、中国スタートアップZ.aiがAI coding tool「ZCode」を発表したと報じました。計画、実装、レビュー、デプロイを1つの環境で支援し、GLM 5.2などのモデルと連携します。料金は米国の主要AI coding toolより低く設定され、CursorやGitHub Copilotへの対抗が意識されています。

### なぜ重要か

前日はClaude Codeの利用禁止や地政学的制限を扱いましたが、今日は逆に中国側の代替ツールが出てくる話です。AI coding toolは性能だけでなく、価格、利用可能地域、データ管理、企業ポリシーで選ばれます。グローバル開発チームでは、国ごとに使えるAI開発環境が分かれる可能性があります。

## 3. AI coding agent、クリーンに見えるGitHubリポジトリ経由でマルウェア実行のリスク

Tom's Hardwareは、Mozillaの0dinチームがAI coding agentを悪用する手法を示したと報じました。見た目は問題のないGitHubリポジトリでも、セットアップ手順や初期化スクリプトを通じてAI agentに不正コマンドを実行させ、開発者端末上でマルウェアを動かす可能性があります。

### なぜ重要か

これはAndroid / モバイル開発者にも直結します。知らないサンプルアプリ、SDK、Gradle plugin、CLIをAIに読ませて「セットアップして」と頼むと、AIが善意で危険なコマンドまで実行する恐れがあります。AI agentを使うなら、リポジトリの出所、実行コマンド、環境変数、APIキー、ローカル権限を分離する設計が必要です。

### 参考リンク

- https://www.tomshardware.com/tech-industry/cyber-security/ai-coding-agents-can-be-tricked-into-installing-malware-via-clean-github-repositories-mozillas-0din-team-shows-how-claude-code-can-be-exploited-by-its-own-helpfulness

## 4. Claude Sonnet 5、チャットからエージェントへの競争軸を強める

TechRadarは、AnthropicのClaude Sonnet 5について、よりエージェント的なモデルとして紹介しました。単に会話で答えるだけでなく、計画、ツール利用、ブラウザやターミナル操作、複雑なタスク実行を重視する方向です。

### なぜ重要か

AIの競争軸は、回答の自然さから「任せられる作業範囲」へ移っています。コードを書く、テストする、PRを作る、ブラウザで調べる、仕様を検証する、といった一連の流れをどこまで安全に任せられるかが実務で重要になります。Android開発でも、単発のComposeコード生成ではなく、Issue単位の実装・検証に移る可能性があります。

### 参考リンク

- https://www.techradar.com/ai-platforms-assistants/claude/claude-sonnet-5-is-here-and-the-most-agentic-sonnet-model-yet-shows-that-the-ai-war-is-shifting-from-chat-to-agents

## 5. GitHub上のAI agent PRを分析するAIDevデータセットが示す実務変化

arXivでは、GitHub上のAI coding agentによるPull Requestを分析するAIDevデータセットが公開されています。OpenAI Codex、Devin、GitHub Copilot、Cursor、Claude Codeなどが作成した約93万件のagentic PRを対象にしており、AIが実際の開発フローへどのように入り込んでいるかを分析できます。

### なぜ重要か

AI coding agentはもうデモだけではなく、PR単位で開発プロセスに入っています。今後は「AIが書いたかどうか」だけでなく、どのAgentが、どの種類の変更を、どのレビュー体制で、どれだけ品質よく入れたかを追跡する必要があります。職務経歴や開発実績の言語化でも、人間の設計判断とAI活用範囲を分けて説明する力が重要になります。

### 参考リンク

- https://arxiv.org/abs/2602.09185
