# 英語表現ミニ講座 2026-07-15

## 今日の表現

**切り分ける**

エンジニアリングでは、問題の原因候補を分離し、調査範囲を狭める意味でよく使います。

## そのまま直訳すると不自然な理由

「切る」と「分ける」をそのまま `cut and separate` とすると、物理的に何かを切断する意味に聞こえます。障害調査では、目的に応じて `isolate`、`narrow down`、`rule out` などを使い分けます。

## 自然な英語表現

- **isolate the cause** — 原因を切り分ける
- **narrow down the issue** — 問題の範囲を絞る
- **rule out ...** — 候補を検証して除外する
- **separate the two problems** — 2つの問題を分けて考える

## 場面別の言い換え

### エンジニアリング

- Let's isolate whether this is a client-side or server-side issue.
- We need to narrow down the conditions that trigger the crash.
- First, let's rule out a network problem.

### ビジネス

- Let's separate the immediate issue from the long-term concern.
- We should narrow down the factors affecting the schedule.

### 日常会話

- Let's figure out which device is causing the problem.
- We need to separate these two issues.

## 短い例文

1. **Let's isolate the cause before changing the code.**  
   コードを変更する前に、原因を切り分けましょう。

2. **I narrowed the issue down to the login flow.**  
   問題をログイン処理まで絞り込みました。

3. **We ruled out the API as the cause.**  
   APIが原因である可能性は除外しました。

4. **Let's separate the UI bug from the performance issue.**  
   UIの不具合と性能問題を分けて考えましょう。

## 今日の5分練習
