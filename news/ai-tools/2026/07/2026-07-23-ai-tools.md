### 1. Claude Code / v2.1.218

- 種別: CLI
- 関連: Claude / Anthropic / Claude Code
- 公開・更新日: 2026-07-22（21:24 UTC / JST 07-23 06:24）※安定版
- 概要: `/code-review` をバックグラウンドのサブエージェント実行に切り替え、Windows のパス化け・MCP 接続診断・複数行ペースト・自動モードの権限判定など、多数の実務不具合を修正した安定版。
- 何が便利か:
  - `/code-review` が**別スレッド（バックグラウンドのサブエージェント）で走る**ため、レビュー出力で会話が埋まらず、直前にスタックしたスラッシュコマンドをレビュー対象として保持できる。
  - `claude mcp list` と `/mcp` に**接続失敗時の HTTP ステータスとエラー本文**が出るようになり、MCP サーバーが繋がらない原因を切り分けやすい。設定値の先頭・末尾の隠れた空白にも警告が出る。
  - Windows で `C:\Users\unicorn` のような **`\u` 始まりのパスが CJK 文字に化けてツール入力から壊れる**問題を修正。Windows 環境の再現性が上がる。
  - `Ctrl+J` で改行をエンコードする端末で**複数行ペーストが 1 行に潰れる**不具合を修正。
  - `/ultrareview` が「review my auth changes」のような**説明的な引数で失敗していた**のを修正（現在のブランチをレビューし、テキストは所見への注記として扱う）。
- 使いどころ:
  - 大きめの PR レビューを回す前に `claude update` で更新し、`/code-review` をバックグラウンドで走らせて会話を汚さずに所見を受け取る。
  - Windows / 企業ネットワーク環境でパス化けや MCP 接続で詰まっていたチームの再現性改善。
- 開発者にとっての価値: レビュー・MCP 運用・Windows 対応という「毎日触る箇所」の摩擦を安定版で減らせる。破壊的変更なしで更新可能。
- 注意点: `/code-review ultra` は非対話セッションで従来サイレントにローカルレビューになっていたが、今回からクラウドレビューを起動する挙動に変わっている点に注意。
- 公式リンク: https://github.com/anthropics/claude-code/releases
- 参考リンク: https://raw.githubusercontent.com/anthropics/claude-code/main/CHANGELOG.md
- メモ: 同日 07-21 の v2.1.217（サブエージェント同時実行上限、MCP 出力のメモリリーク修正など）は 07-22 記事で採用済み。今回は 07-22 の v2.1.218 のみを新規採用。

#### 使用例

```bash
# 更新
claude update

# 現在のブランチの変更をバックグラウンドのサブエージェントでレビュー
claude
> /code-review

# MCP サーバーの接続失敗理由（HTTP ステータス）を確認
claude mcp list
```

### 2. Gemini CLI / v0.52.0（安定版）

- 種別: CLI
- 関連: Google AI / Gemini CLI
- 公開・更新日: 2026-07-22（20:51 UTC / JST 07-23 05:51）※安定版
- 概要: コンテキスト構築の精度改善（CI 設定ファイルの除外、JSON / IPYNB の LLM 補正バイパス）と、ワークスペース信頼・タスク分離による RCE 対策を盛り込んだ安定版。
- 何が便利か:
  - **CI 設定ファイルをコンテキストから除外**し、リポジトリの本質と関係ないノイズが会話に混ざりにくくなった。
  - **JSON / IPYNB ファイルは LLM 補正をバイパス**するため、構造化データやノートブックが意図せず整形・改変されるのを防げる。
  - **ワークスペース信頼とタスク分離を強制**し、信頼していないディレクトリからの RCE（リモートコード実行）経路を塞ぐセキュリティ修正。
- 使いどころ:
  - Notebook（.ipynb）や設定 JSON を多く含むリポジトリで、意図しない自動整形を避けたいとき。
  - 未検証のリポジトリを Gemini CLI で開く前提のワークフローでのセキュリティ強化。
- 開発者にとっての価値: 生成物の副作用（勝手な書き換え）を減らしつつ、信頼境界を明確にできる安定版アップデート。
- 注意点: 同時期に v0.53.0-preview.0（07-22 20:10 UTC）も出ているが**プレリリース**。安定運用では v0.52.0 を使う。
- 公式リンク: https://github.com/google-gemini/gemini-cli/releases
- 参考リンク: https://github.com/google-gemini/gemini-cli
- メモ: 07-21 記事時点の最新は v0.51.0（セキュリティ修正）。今回は次の安定版 v0.52.0 を新規採用。

#### 使用例

```bash
# npm 経由で更新
npm install -g @google/gemini-cli@latest

# バージョン確認
gemini --version
```

### 3. Vercel AI SDK / ai@7.0.35（+ 6.0.234・5.0.219）

- 種別: OSS / SDK（TypeScript）
- 関連: Vercel AI SDK
- 公開・更新日: 2026-07-22（19:07〜19:10 UTC / JST 07-23 04:07〜04:10）※安定版
- 概要: ストリーミングのエラーハンドリングとタイムアウト制御、メディアタイプ判定のパフォーマンスを改善したメンテナンス安定版。3 系統（v7 / v6 / v5）が同時リリース。
- 何が便利か:
  - **レスポンスのパイプ処理を Promise で返す**ようになり、ストリームの読み取り・書き込みエラーを呼び出し側で `try/catch`（`.catch`）で確実に捕捉できる。取りこぼしていたストリーム系エラーをハンドリングしやすい。
  - ストリーミング生成に**ステップごとの「最初のコンテンツ到達までのタイムアウト」**を追加。モデルが無音のまま固まるケースを検知・打ち切りできる（v7.0.35）。
  - base64 添付が **`ID3` で始まると添付全体をデコードしていたメディアタイプ判定を O(1) に最適化**（v6.0.234 / v5.0.219）。大きな添付での無駄な処理を削減。
- 使いどころ:
  - ストリーミング応答を UI に流すアプリで、途中のネットワーク断・書き込みエラーを握りつぶさずに扱いたいとき。
  - 音声・大容量添付を扱う経路で、メディアタイプ判定の CPU コストを抑えたいとき。
- 開発者にとっての価値: プロダクションのストリーミング信頼性（エラー捕捉・タイムアウト）と添付処理の効率が上がる。メジャー変更なしで取り込める。
- 注意点: v7 / v6 / v5 が併走しているため、利用中のメジャーに合わせて該当パッチ（7.0.35 / 6.0.234 / 5.0.219）を選ぶ。関連して `@ai-sdk/gateway`（4.0.27 / 3.0.156 / 2.0.118）も更新。
- 公式リンク: https://github.com/vercel/ai/releases
- 参考リンク: https://ai-sdk.dev/
- メモ: 07-22 記事は `ai@7.0.34`（MP4 音声の ftyp 検出など）までを採用。今回は 07-22 リリースの 7.0.35 / 6.0.234 / 5.0.219 を新規採用。

#### 使用例

```bash
# 利用中のメジャーに合わせて更新（例: v7 系）
npm install ai@7.0.35
```
