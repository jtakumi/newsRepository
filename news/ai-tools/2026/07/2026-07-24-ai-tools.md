### 1. Ollama / v0.32.3

- 種別: OSS / CLI（ローカル LLM ランタイム）
- 関連: Ollama / ローカルLLM / Claude Code / Anthropic
- 公開・更新日: 2026-07-23（00:44 UTC / JST 07-23 09:44）※安定版
- 概要: モデルダウンロードの stall 修正を中心に、Claude Code 連携・Anthropic thinking ストリーム・複数 GPU バックエンドの対応を強化した安定版。直前の v0.32.2 は取り下げ（withdrawn）扱いで、v0.32.3 以降への更新が案内されている。
- 何が便利か:
  - **`ollama pull` がデータ送信前に固まる（stall）不具合を修正**。大きめのモデル取得が途中で止まる問題を解消。
  - **Claude Code Channels 連携を復旧**し、**Anthropic の thinking ストリームの不具合を修正**。ローカルモデルをエージェント／思考表示付きで使う経路が安定。
  - **GPU 対応を拡大**: Windows ARM64 で CUDA が使えるように、B200 GPU を CUDA 12 でサポート、Linux の CUDA/ROCm 統合 GPU でメモリ消費を削減。
  - GLM のツール呼び出しの不具合修正、Laguna 2.1 系のチャット／thinking／ツール呼び出し対応（Metal 推論の修正込み）、MLX・llama.cpp エンジン更新。
- 使いどころ:
  - ローカルモデルの取得が途中で止まって困っていた開発マシンの復旧。
  - Claude Code などのエージェントからローカルモデル（thinking 表示あり）を呼ぶワークフローの安定化。
  - Windows ARM64 ノートや B200 環境で GPU 推論を回したいとき。
- 開発者にとっての価値: ローカル LLM の「取得できない・GPU が効かない・エージェント連携が不安定」という基礎の詰まりをまとめて解消できる安定版。オフライン検証や機密データのローカル処理の土台が固まる。
- 注意点: 直前の v0.32.2 はプレリリースとして取り下げられているため、v0.32.2 を入れていた場合は v0.32.3 以降へ更新すること。

- メモ: 07-23 記事時点では v0.32.2（プレリリース）扱いで見送っていた。今回は安定版 v0.32.3 を新規採用。

### 3. Vercel AI SDK / ai@7.0.36

- 種別: OSS / SDK（TypeScript）
- 関連: Vercel AI SDK
- 公開・更新日: 2026-07-23（14:33 UTC / JST 07-23 23:33）※安定版
- 概要: ツール承認署名（`experimental_toolApprovalSecret`）の HMAC ペイロード直列化における署名衝突の脆弱性を修正したセキュリティパッチ。関連パッケージ（react / vue / svelte / rsc / tui / workflow など）も同時更新。
- 何が便利か:
  - **ツール承認署名の直列化の脆弱性を修正**。従来は各フィールドを改行（`\n`）で連結していたため、`toolName` や `toolCallId` 自体に改行が含まれると、異なるフィールドの組み合わせが同一バイト列になり、**署名が別のタプルに対して通ってしまう**恐れがあった。
  - **バージョン付きドメイン分離を前置した `JSON.stringify` ベースの単射的直列化**に変更し、区切り文字・制御文字を適切にエスケープして衝突を排除。
  - **後方互換を維持**。どのフィールドにも `\n` を含まない従来形式の署名は引き続き有効なので、アップグレード中の保留承認が壊れない。
- 使いどころ:
  - Human-in-the-loop でツール実行を承認するフロー（`experimental_toolApprovalSecret` を使う構成）を本番で運用している場合は、早めに更新して署名の健全性を担保する。
- 開発者にとっての価値: ツール承認という「実行を止める安全弁」の署名検証が確実になる。エージェントにツール実行権限を与えるアプリのセキュリティ強化として重要。
- 注意点: 影響するのは `experimental_toolApprovalSecret` を使う構成。利用中のメジャー（v7 / v6 / v5）に合わせて該当パッチを確認すること（本記事は v7 系の 7.0.36 を対象）。
- 公式リンク: https://github.com/vercel/ai/releases/tag/ai%407.0.36
- 参考リンク: https://ai-sdk.dev/
- メモ: 07-23 記事は `ai@7.0.35`（ストリームのエラーハンドリング・タイムアウト）までを採用。今回は 07-23 リリースのセキュリティ修正 7.0.36 を新規採用。

#### 使用例

```bash
# v7 系を更新
npm install ai@7.0.36
```

### 4. Cline Code Desktop / v0.0.2・v0.0.3

- 種別: デスクトップアプリ（macOS）
- 関連: Cline
- 公開・更新日: 2026-07-22（v0.0.2: 00:08 UTC / JST 09:08、v0.0.3: 00:37 UTC / JST 09:37）
- 概要: これまで VS Code 拡張として提供されてきた Cline を、**単体の macOS デスクトップアプリとして初公開**したもの。v0.0.2 が初回公開、v0.0.3 は「Thinking」表示まわりの UI 調整。
- 何が便利か:
  - **VS Code 拡張なしで Cline エージェントのセッションを実行・確認できる**独立アプリ。Apple Silicon / Intel 両対応で、署名・notarize 済み。
  - **自動更新に対応**（起動時と 2 時間ごとにチェック、バックグラウンドでダウンロードし、ワンクリック再起動で反映）。無視した更新も次回起動時に適用。
  - v0.0.3 で、チャット中の推論表示を単に「Thinking」に整理する UI 調整。
- 使いどころ:
  - VS Code を常用しない、あるいはエディタと切り離してエージェント作業をしたいときに、Cline を単体アプリとして試す。
  - macOS でエージェントのセッションを独立ウィンドウで回したいとき。
- 開発者にとっての価値: Cline を「エディタ拡張」から「独立アプリ」として使える選択肢が増えた。エージェント作業をエディタから分離したいワークフローに合う。
- 注意点: まだ **v0.0.x の初期段階**で、機能は最小限。現時点では macOS のみ。本格運用よりは評価用途向き。
- 公式リンク: https://github.com/cline/cline/releases/tag/desktop-v0.0.2
- 参考リンク: https://github.com/cline/cline/releases
- メモ: これまでの AI Tools News では Cline は CLI / VS Code 拡張を扱っていた。デスクトップアプリの初公開は未掲載のため新規採用。
