- **Claude Code v2.1.215（07-19 02:56 UTC）** で、**`/verify` と `/code-review` スキルが自動実行されなくなり、明示的に `/verify` / `/code-review` を打ったときだけ動く**オプトインに変更。エージェントが勝手にレビュー/検証を走らせてトークン・時間を消費する挙動を避けたい人向けのピンポイント修正。

## 今日の注目

### 1. Cline / CLI v3.0.45・v3.0.46・SDK v0.0.65

- 種別: CLI / SDK（OSS）
- 関連: Cline
- 公開・更新日: 2026-07-19（SDK v0.0.65 は 04:02 UTC、CLI v3.0.45 は 04:10 UTC、CLI v3.0.46 は 06:26 UTC）
- 概要: Cline の CLI / SDK が同日に連続更新。**CLI v3.0.45** の目玉は**インストールフットプリントの大幅削減**で、Claude Code / Codex のプロバイダーを**オプショナル（オンデマンド読み込み）**に切り替え、`npm i -g cline` を **約 640MB から 285MB** へ縮小。あわせて **Kimi K3** を ClinePass のモデルとして追加、**期限切れ OAuth 資格情報のリフレッシュ後に run を 1 回リトライ**、チーム実行の改善（teammate に spawn ツールを非公開化、失敗した teammate run を「失敗」として正しく報告）、`hub status` 出力へのバージョン表示などを含む。**CLI v3.0.46** は、Cline API の **本物の `insufficient_credits`（402）エラーを確実に認識**し、汎用エラーではなく**クレジット追加 UI を表示**するよう修正。**SDK v0.0.65** は、上記のプロバイダー分離・Kimi K3・OAuth リフレッシュ・チーム実行制限を SDK 側でも提供し、新しい `@cline/ui` エージェントチャットコンポーネント（Storybook 対応）を追加。
- 何が便利か: グローバルインストールが **半分以下のサイズ**になり、CI イメージやローカル環境のディスク/取得時間が軽くなる。プロバイダーは必要になったときだけ落ちてくるため、Claude Code / Codex を使わない構成なら無駄がない。OAuth 期限切れで run が落ちていたケースが**自動リトライ**で拾えるようになり、クレジット切れも**理由が明確に**表示される。
- 使いどころ: CI/コンテナで Cline CLI を都度インストールして回す構成、ディスクの限られた環境、ClinePass 経由で複数モデル（Kimi K3 など）を切り替える運用、長時間バッチで OAuth トークンが切れやすい構成。
- 開発者にとっての価値: 07-17 分（OS 証明書ストア自動取り込み・BOM 対応）に続く**導入・運用摩擦の低減**。特にインストールサイズの半減は CI のキャッシュ効率とコールドスタートに効く、地味だが効果の分かりやすい改善。
- 注意点: プロバイダーがオプショナル化されたため、**Claude Code / Codex プロバイダーを使う場合は初回にオンデマンドで追加取得が走る**点に留意（オフライン前提の環境では事前取得を検討）。Kimi K3 は ClinePass 経由の追加であり、対応可否は契約/プランに依存。
- 公式リンク: https://github.com/cline/cline/releases/tag/cli-v3.0.45
- 参考リンク: https://github.com/cline/cline/releases , https://github.com/cline/cline
- メモ: 「毎回グローバルインストールする CI で Cline が重い」構成なら、v3.0.45 以降でイメージ肥大とダウンロード時間が目に見えて改善する。

#### 使用例

```bash
# グローバルインストール（v3.0.45 以降はサイズが約 285MB に）
npm i -g cline
cline --version            # 3.0.46 を確認

# ハブ状態にバージョン情報が出るようになった
cline hub status
```

### 2. LiteLLM / v1.93.0（安定版）

- 種別: OSS / API ゲートウェイ（プロキシ）
- 関連: LiteLLM / MCP / OpenRouter 的なマルチプロバイダー基盤
- 公開・更新日: 2026-07-19（v1.93.0、07:57 UTC）※安定版。あわせて v1.92.1 / v1.91.4 / v1.90.6 のバックポートも同日公開
- 概要: 前回窓では rc/dev 段階だった 1.93 系が**安定版 v1.93.0** として公開。主な内容は、(1) **MCP**: OAuth2 トークン交換（`entra_obo` プロファイル）による on-behalf-of 認証、`true_passthrough` / `oauth_delegate` などの認証タイプ追加、ツールルーティングの alias/表示名対応と REST フィルタ、(2) **ストリーミング/Messages API**: ストリーミング `/v1/messages` 応答で**プロバイダーのレスポンスヘッダーを転送**、初回チャンクへの `completion_start_time` 付与、OpenAI 互換ストリームでの in-body エラー表面化、Bedrock の `message_stop` 欠落時のエラー処理改善、(3) **ガードレール/セキュリティ**: ガードレール応答に含まれる**資格情報を永続化前にマスキング**、**`Bearer` プレフィックス付き API キーを spend ログでハッシュ化**、Content Filter の `pre_mcp_call` 対応、(4) **プロバイダー/モデル**: **x.ai Grok-4.5 の価格**、**Meta Model API（muse-spark-1.1）**、**Google Cloud Speech-to-Text Chirp 3** 文字起こし、Azure の data-zone / long-context（GPT-5.4/5.5）価格の追加。
- 何が便利か: 複数プロバイダーを 1 つの OpenAI 互換エンドポイントに束ねる LiteLLM で、**MCP の企業認証（Entra OBO）**や**ストリーミング時のヘッダー透過**といった、実運用で詰まりやすい部分が安定版に載る。ログ/ガードレール応答からの**資格情報漏えい対策**が入り、監査観点でも扱いやすくなる。新モデル（Grok-4.5 ほか）の**価格/コスト計算**が最新化される。
- 使いどころ: 社内で LiteLLM Proxy をマルチプロバイダーのゲートウェイとして立てている構成、MCP ツールを企業 IdP（Entra）認証で通したいケース、ストリーミングでプロバイダー固有ヘッダーを下流に渡したい構成、spend ログを監査に使う運用。
- 開発者にとっての価値: 前回まで rc/dev のみで安定運用に載せづらかった機能群が、**安定版として一括で取り込める**。特にコスト計算・ガードレールのマスキング・MCP 認証は、本番ゲートウェイの信頼性と安全性に直結する。
- 注意点: 機能が広範なぶん、MCP 認証やガードレール周りは**自環境での挙動確認**を推奨。helm チャートが `helm/` 新ディレクトリ構成へリファクタされたため、Kubernetes デプロイ勢はチャートパスの変更に注意。
- 公式リンク: https://github.com/BerriAI/litellm/releases/tag/v1.93.0
- 参考リンク: https://github.com/BerriAI/litellm/releases , https://docs.litellm.ai/
- メモ: 1.93 系を rc で見送っていたなら、v1.93.0 安定版が本採用の分かれ目。監査ログに API キーが平文で残るのを避けたい構成では、マスキング/ハッシュ化の修正だけでも更新価値がある。

### 3. Claude Code / v2.1.215

- 種別: CLI
- 関連: Claude / Anthropic / Claude Code
- 公開・更新日: 2026-07-19（v2.1.215、02:56 UTC）
- 概要: Anthropic のコーディングエージェント CLI が小幅更新。変更点は 1 件で、**`/verify` と `/code-review` のスキルを Claude が自発的に実行しなくなり、使いたいときに `/verify` / `/code-review` を明示的に呼び出す**方式（オプトイン）に変更。直前の v2.1.214（07-18）で入った権限ハードニング一式の後の、挙動を予測しやすくするための調整。

- 開発者にとっての価値: 07-19 記事の v2.1.212/214（権限ハードニング・長時間実行制御）に続く**挙動の予測可能性**の改善。`claude update` だけで反映でき、追加設定は不要。
