## 今日の注目

### 1. Claude Code / v2.1.210

- 種別: CLI
- 関連: Claude / Anthropic / Claude Code
- 公開・更新日: 2026-07-14 23:45 UTC（日本時間 2026-07-15 08:45）
- 概要: Anthropic のコーディングエージェント CLI のメンテナンス/UX リリース。主な内容は (1) 折りたたまれたツール要約行に**ライブ経過時間カウンター**を表示し、長時間処理が動いていることを可視化、(2) 非推奨のパーミッションルール `Write` / `NotebookEdit` / `Glob` に対する**起動時警告**（`Edit` / `Read` への移行を促す）、(3) `isolation: 'worktree'` のサブエージェントが**メインリポジトリに対して git を変更するコマンドを実行してしまう不具合の修正**、(4) `ultracode` キーワードのオプトインが **webhook・PR コメントなど人間由来でない入力で誤発火する問題の修正**、(5) プラグイン MCP サーバーが再同期中に破棄されない、スクリーンリーダーモードでのパーミッションモード変更のアナウンス、`claude attach` の「job not found」修正、外部エディタへペーストマーカーが漏れない修正など、合計 40 件超のバグ修正・改善。
- 何が便利か: `claude update` するだけで、**バックグラウンド/サブエージェント運用の地雷（worktree でのメインリポジトリ汚染、webhook からの `ultracode` 誤発火）**が解消される。経過時間カウンターで「固まっているのか動いているのか」が一目で分かる。
- 使いどころ: `claude agents` やワークフローでサブエージェントを並行実行する運用、CI・webhook 経由で Claude Code を起動する自動化、支援技術を使う開発者。
- 開発者にとっての価値: 前日記事の v2.1.208/209（アクセシビリティ・入力体験・バックグラウンド安定化）に続く**安定化リリース**。新機能より、エージェントを実運用に載せたときの取りこぼしを潰す修正が中心で、常用者ほど効く。
- 注意点: 目玉は新機能ではなく**バグ修正・安定性・警告の追加**。非推奨パーミッションルールを使っている設定は起動時に警告が出るため、`Edit` / `Read` への置き換えを検討するとよい。
- 公式リンク: https://github.com/anthropics/claude-code/releases/tag/v2.1.210
- 参考リンク: https://github.com/anthropics/claude-code/releases
- メモ: `isolation: 'worktree'` の git 変更漏れ修正は、並列でファイルを触るサブエージェントを使う構成では取り込む価値が高い。webhook 起動の自動化がある場合も `ultracode` 誤発火修正のため更新推奨。

#### 使用例

```bash
# 更新して最新版を確認
claude update
claude --version          # 2.1.210 を確認

# 非推奨パーミッションルールの警告が出たら Edit / Read へ置き換える
#   （settings.json の "Write" / "NotebookEdit" / "Glob" を見直す）
```

### 2. Cline / CLI v3.0.41・SDK v0.0.61

- 種別: OSS（VS Code 拡張 + CLI + SDK）
- 関連: Cline / その他（OpenAI 互換エンドポイント・ローカル LLM）
- 公開・更新日: 2026-07-15（SDK v0.0.61 は 17:57、CLI v3.0.41 は 18:09）
- 概要: OSS コーディングエージェント Cline の CLI と SDK が同日更新。共通の改善として (1) **コンテキスト圧縮（compaction）の進捗ステータス表示**（CLI は TUI、SDK は実行中に進捗を報告）、(2) プロバイダー設定時に **OpenAI 互換エンドポイントからモデル ID を自動サジェスト**、(3) ワークスペースの **git 情報（ブランチ/リモート）をセッション跨ぎで永続化・再取得**、(4) **ターン実行中の圧縮を抑止**、(5) 無害な git 状態を初期化エラーと誤検知しないよう修正。SDK 側はさらに Plan/Act モードのガイダンスをシステムプロンプトに統合、SDK 編集ツールのエディタ差分表示を復元、共有 UI テーマパッケージ `@cline/ui` を導入。
- 何が便利か: **ローカル/セルフホストや OpenAI 互換 API を使う際に、モデル ID を手打ちせず候補から選べる**。長い作業でも compaction の進捗が見えるので「圧縮で止まっているのか」が分かりやすい。git 情報が永続化され、セッション再開時の文脈把握が安定する。
- 使いどころ: VS Code 拡張だけでなく**ターミナルの CLI やスクリプトから Cline を回す**構成、Ollama / vLLM / OpenRouter など OpenAI 互換エンドポイントを差し替えて使う開発、長時間のリファクタ/移行タスク。
- 開発者にとっての価値: モデルの自前ホスティングや API プロバイダーの乗り換えが多い開発者にとって、**設定の摩擦が下がる**。CLI・SDK が同時に更新されるため、エディタ外の自動化に組み込みやすい。
- 注意点: 今回は圧縮・設定・git まわりの体験改善が中心で、大きな新機能ではない。SDK は v0.0.x 系でインターフェースが変わりうる。
- 公式リンク: https://github.com/cline/cline/releases/tag/cli-v3.0.41
- 参考リンク: https://github.com/cline/cline/releases/tag/sdk/sdk/v0.0.61 , https://github.com/cline/cline/releases
- メモ: OpenAI 互換エンドポイントのモデル ID サジェストは、ローカル LLM を頻繁に切り替える人には地味に効く。CLI 版は TUI での進捗可視化が入り、非対話的な自動実行との相性も改善。

#### 使用例

```bash

npm install -g @cline/cli    # 環境に応じてインストール方法は要確認
cline --version              # 3.0.41 を確認

# OpenAI 互換エンドポイント（例: ローカル LLM）を設定
#   プロバイダー設定時にモデル ID が候補としてサジェストされる
```
