### 1. Claude Code / v2.1.212・v2.1.214

- 関連: Claude / Anthropic / Claude Code
- 公開・更新日: 2026-07-17（v2.1.212、00:26 UTC）／ 2026-07-18（v2.1.214、01:20 UTC）
- 概要: Anthropic のコーディングエージェント CLI が連続更新。**v2.1.214** は権限チェックのハードニングが主軸で、(1) `Edit(src/**)` のような**単一セグメント `dir/**` allow ルールがツリー内の任意の `dir/` への書き込みを自動承認**していた問題を、`<cwd>/dir` のみに一致するよう修正、(2) **Windows PowerShell 5.1 セッションでの権限チェックバイパス**、(3) **ファイルディスクリプタのリダイレクト形式**や **1 万文字超のコマンド**（今後は常にプロンプト）、`[[ ]]` 内の zsh 変数サブスクリプト、危険なオプションを含みうる一部 `help`/`man`、`docker` のデーモンリダイレクトフラグなどを**取りこぼしていた権限判定を是正**。あわせて (4) 高度に悪質・脱獄的なセッションを終了できる **EndConversation ツール**、(5) 沈黙しがちな長時間ツール呼び出し向けの**定期プログレスハートビート**、(6) OpenTelemetry ログへの `message.uuid` / `client_request_id` / `tool_source` 属性追加と `CLAUDE_CODE_OTEL_CONTENT_MAX_LENGTH`、(7) Windows PowerShell ツールの UTF-8/UTF-16LE エンコーディングや `where.exe`/`fc.exe`/`diff.exe` の誤エラー、(8) 企業プロキシ配下 Windows の "Socket is closed"、`stream-json` 出力の終了時切り捨て、アイドル後のバックグラウンドデーモン不具合などを多数修正。**v2.1.212** は **`/fork` が会話を新しいバックグラウンドセッションへ複製**（従来のセッション内サブエージェントは `/subtask` に分離）、**`claude auto-mode reset`（確認付き）**、**WebSearch のセッション上限（既定 200）**・**サブエージェント生成のセッション上限（既定 200）**、**2 分超の MCP ツール呼び出しの自動バックグラウンド化**、削除済みも含む過去セッションを選べる **`/resume` ピッカー**、**プランモードがファイル変更コマンドを権限プロンプトなしで自動実行していた問題**の修正などを追加。
- 何が便利か: 自動実行（auto モード）・`claude agents`・CI で回すほど効く**権限判定の穴ふさぎ**が中心で、意図しない書き込みやコマンド自動承認のリスクが下がる。WebSearch・サブエージェントの**暴走上限**と MCP の**自動バックグラウンド化**で、長時間・多段のエージェント実行が安定・予測可能に。

- 注意点: 目玉は新機能というより**権限ハードニングと運用制御**。`dir/**` の解釈変更（allow ルール・hook 条件とも `<cwd>/dir` のみ一致）で、これまで広く一致していた既存ルールの**挙動が変わる**可能性があるため、自前の allow/hook 設定は見直すと安全。上限（WebSearch 200 / サブエージェント 200）は既定値で、必要に応じて調整前提。
