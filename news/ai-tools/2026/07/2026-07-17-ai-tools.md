### 1. Claude Code / v2.1.211

- 種別: CLI
- 関連: Claude / Anthropic / Claude Code
- 公開・更新日: 2026-07-15 23:02 UTC（JST 2026-07-16 08:02）
- 概要: Anthropic のコーディングエージェント CLI の最新版。セキュリティ面では **permission preview が双方向オーバーライド・ゼロ幅・そっくりな（look-alike）引用符文字を無害化しない不具合**を修正し、同期される skill/plugin ディレクトリ命名も強化。運用面では **「always allow」権限ルールをリポジトリルートに保存**するよう変更し、git worktree で与えた承認がセッション・worktree をまたいで持続するように。新機能として **`--forward-subagent-text` フラグ／`CLAUDE_CODE_FORWARD_SUBAGENT_TEXT` 環境変数**で subagent のテキストと思考を stream-json 出力に含められるようになり、整数系の環境変数（タイムアウト・トークン予算・リトライ回数）が **`1e6` や `64_000`** といった指数・桁区切り表記を受け付けるようになった。Vim モードの `s` / `S`（文字・行の置換）が NORMAL モードで動作。バグ修正は多数で、**wake-from-sleep 後に並行セッションが一斉ログアウトする**不具合、**アイドルな web セッション復帰後に plugin MCP サーバーが再接続しない**不具合、Vertex/Bedrock 起動時に既定 Opus を試みる不具合、モデル上書き指定の subagent が親モデルに戻る不具合、`/loop` がセッションを `/resume` から隠す不具合、`/clear` がコストカウンタをリセットしない不具合、Bedrock/Vertex/Mantle/Foundry のプロンプトキャッシュ回帰などを解消。
- 何が便利か: **worktree でブランチ並行開発**する際に、承認のたびに聞かれ直す摩擦が減る（承認がルートに保存され持続）。stream-json パイプラインを組む自動化では、**subagent の中身まで観測できる**ようになり CI やログ収集の解像度が上がる。トークン予算を `1e6` のように書けるので設定ミスも減る。
- 使いどころ: 複数 worktree でのブランチ並行作業、`claude -p` / stream-json での自動化・ログ収集、plugin MCP を常用する構成、支援技術（スクリーンリーダー）利用者。
- 開発者にとっての価値: 前系（v2.1.208/209 の入力体験・バックグラウンド安定化）に続く**権限まわりのセキュリティ強化と worktree 運用の実務改善**。日常的に回すほど効く修正が中心。
- 注意点: 目玉は大きな新 API ではなく**セキュリティ・権限・安定性**の改善。`--forward-subagent-text` は subagent の思考まで出力に含めるため、ログの取り扱いには注意。
- 公式リンク: https://github.com/anthropics/claude-code/releases/tag/v2.1.211
- 参考リンク: https://github.com/anthropics/claude-code/releases , https://docs.claude.com/en/docs/claude-code
- メモ: permission preview の Unicode 無害化は、そっくり文字を使った承認プロンプトの誤読を防ぐ地味だが重要な修正。承認の判断を人が目視する運用ほど効く。

#### 使用例

```bash

claude update
claude --version          # 2.1.211 を確認

# subagent のテキスト・思考を stream-json に含める
claude -p "run tests and summarize" --forward-subagent-text --output-format stream-json

# 環境変数を指数・桁区切り表記で指定
export MAX_THINKING_TOKENS=64_000
```

### 2. OpenAI Codex CLI / v0.144.5

- 種別: CLI
- 関連: OpenAI / Codex
- 公開・更新日: 2026-07-16（安定版、02:54）
- 概要: OpenAI のコーディングエージェント CLI の安定版。**危険コマンド検知（dangerous-command detection）を強化**し、**強制 `rm`（forced `rm`）の亜種をより多く検出**するようになった。あわせて**コマンドが拒否されたときの理由をより明確に**返すように改善。直前の v0.144.4（07-14）はユーザー向け変更なしのメンテナンスリリースだったため、本記事では v0.144.5 が新規分。
- 何が便利か: エージェントにシェル実行を任せる際、`rm -rf` 系の破壊的コマンドの取りこぼしが減り、**ブロックされた理由がわかる**ので「なぜ止まったか」を追いやすくなる。承認フローの誤爆・見逃しの両方を抑えられる。
- 使いどころ: Codex にファイル操作やクリーンアップを任せる自動化、CI での自律実行、権限を絞って走らせたい共有環境。
- 開発者にとっての価値: エージェント CLI の**ガードレールの精度向上**は、自律実行の信頼性に直結する。拒否理由が明確になることで、正当なコマンドが誤ってブロックされた際の切り分けも速くなる。
- 注意点: 本リリースは新機能ではなく**安全検知の強化**が主眼。検知はあくまで補助であり、破壊的操作の最終的な防御（バックアップ・権限分離）を置き換えるものではない。v0.145.0-alpha 系は**プレリリース**のため安定運用は v0.144.5 まで。
- 公式リンク: https://github.com/openai/codex/releases/tag/rust-v0.144.5
- 参考リンク: https://developers.openai.com/codex/changelog , https://github.com/openai/codex/releases
- メモ: 前系（v0.144.2/3 の Guardian 自動レビュー回帰ロールバック）に続く安全側の連続パッチ。エージェントにシェルを任せる構成では最新の安定版へ追従する価値がある。

#### 使用例

```bash
codex --version            # 0.144.5 を確認（未満なら更新）
# 危険コマンド検知が働くと、拒否理由つきでブロックされる
```

### 3. Ollama / v0.32.1

- 種別: OSS / ローカル LLM ランタイム
- 関連: Ollama / ローカルLLM / Gemma
- 公開・更新日: 2026-07-16（安定版）
- 概要: ローカル LLM ランタイム Ollama の安定版。**Gemma 4 のツール呼び出しとマルチターン推論を改善**し、ツール応答の継続（tool-response continuations）の信頼性を上げた。**リクエストをまたいでメモリ使用量を増やしうる MLX モデルのキャッシュリーク**を修正し、キャッシュスナップショット性能も改善。**MLX テキストモデルのロードが `OLLAMA_LOAD_TIMEOUT` を尊重**するようになった。07-11 の v0.32.0 で入った**対話型エージェント体験**のフォローアップとして、agent の web 検索/取得が必要時に **`ollama signin` を促す**ようになり、**対話型エージェントにカレントディレクトリを渡してプロジェクト文脈を強化**。`ollama launch` で非推奨モデルを `--model` 指定した際の model picker 表示も修正。VS Code 拡張のドキュメントも更新。
- 何が便利か: **ローカルの Gemma 4 でツール呼び出しを含むエージェント**を組む際の安定性が上がる。Apple Silicon（MLX）環境での**メモリリーク解消**は長時間稼働で効く。対話型エージェントが cwd を認識するため、プロジェクト内でそのまま作業を任せやすい。
- 使いどころ: ローカル完結でエージェント/ツール実行を試したい個人開発、Apple Silicon での長時間ローカル推論、`ollama` の対話型エージェントをリポジトリ直下で使う運用。
- 開発者にとっての価値: クラウドに出さずに**ツール呼び出し付きエージェントをローカルで検証**できる土台が固まる。v0.32.0 の対話型エージェントを実運用に載せるうえでの信頼性向上パッチ。
- 注意点: MLX 関連の修正は主に Apple Silicon 環境向け。対話型エージェントの web 検索/取得は `ollama signin` が前提になる場合がある。非推奨モデル（CodeLlama, Qwen2.5, Llama 3.x など）は v0.32.0 で警告対象。
- 公式リンク: https://github.com/ollama/ollama/releases/tag/v0.32.1
- 参考リンク: https://github.com/ollama/ollama/releases , https://ollama.com/
- メモ: v0.32.0（07-11）の対話型エージェントは 48h 窓外のため前回まで本文未採用だったが、v0.32.1 で cwd 認識・安定化が入り実用度が上がった。ローカルエージェントを試すなら v0.32.1 以降が無難。

#### 使用例

```bash
# 更新して対話型エージェントを起動（cwd をプロジェクト直下に）
ollama --version          # 0.32.1 を確認
cd your-project && ollama
# web 検索/取得を使う場合は認証
ollama signin
```

### 4. Cline / CLI v3.0.42・SDK v0.0.62（＋ v3.0.41 / v0.0.61）

- 種別: CLI / SDK / OSS
- 関連: Cline / Ollama
- 公開・更新日: 2026-07-15（CLI v3.0.41 / SDK v0.0.61）〜 2026-07-16（CLI v3.0.42 / SDK v0.0.62）
- 概要: オープンソースのコーディングエージェント Cline の CLI/SDK 連続更新。**v3.0.41 / v0.0.61（07-15）** は、**コンテキスト圧縮（compaction）の進捗をターミナル/SDK に表示**、**OpenAI 互換エンドポイントからモデル ID をサジェスト**、**セッションをまたいでワークスペースの git 情報を保持**、SDK ツール向けのエディタ差分表示を復元。さらに **Plan/Act モードのガイダンスをシステムプロンプトに追加**し、セッション途中のモード切替が見えるように。**アクティブなターン中の compaction を抑止**し、ターミナルタイトル更新時のクラッシュも修正。続く **v3.0.42 / v0.0.62（07-16）** は、**Ollama ネイティブ API ルーティングを復元**して**コンテキストウィンドウとタイムアウト設定が再び効く**ように修正し、hub ツールコンテキストからテレメトリを除去。
- 何が便利か: 長いセッションで**いつ compaction が走っているか**が見えるため、コンテキスト圧縮による挙動変化を把握しやすい。**Ollama をバックエンドにした際の context window/timeout 設定が復活**したので、ローカルモデル運用の取り回しが戻る。OpenAI 互換エンドポイント利用時にモデル ID を手打ちしなくてよい。
- 使いどころ: Cline を CLI/SDK で自動化に組み込む構成、Ollama などローカル/OpenAI 互換バックエンドを使う環境、長時間セッションで compaction を多用するタスク。
- 開発者にとっての価値: エージェント CLI/SDK を日常的に回すほど効く**可観測性（compaction 進捗）とバックエンド設定の整合**の改善。git 情報のセッション跨ぎ保持は、再開時の文脈復元に効く。
- 注意点: 主にバグ修正・運用改善で大きな新機能ではない。Ollama ルーティングの修正は v3.0.42 / v0.0.62 で入るため、Ollama 併用時はこのバージョン以降へ。CLI と SDK、GUI（VS Code 拡張）でバージョン体系が別。
- 公式リンク: https://github.com/cline/cline/releases
- 参考リンク: https://docs.cline.bot/
- メモ: 前系（CLI v3.0.40 の OAuth 手動キー逃げ道）に続く運用摩擦の低減。ローカル（Ollama）+ Cline の構成を使うなら v3.0.42 以降が安定。

#### 使用例

```bash
npm install -g @cline/cli   # または bun add -g @cline/cli
cline --version             # 3.0.42 を確認
# Ollama をバックエンドにする場合、context window/timeout 設定が再び反映される
```
