### 1. Claude Code / v2.1.203・v2.1.204

- 種別: CLI
- 関連: Anthropic / Claude Code
- 公開・更新日: 2026-07-07（2.1.203）〜 2026-07-08（2.1.204）
- 概要: ログイン期限が近いと再認証を促す警告を追加。手動許可モード時にフッターへ灰色の ⏸ バッジを表示し、現在の許可モードが常に見えるように。セッションの追加作業ディレクトリを MCP の `roots/list` に反映し、変更時は `notifications/roots/list_changed` を送出。macOS のバックグラウンドセッション起動が誤った低メモリ判定で 15〜20 秒停止する回帰や、worktree 分離サブエージェントが親チェックアウトでコマンドを実行してしまう不具合など、多数のバックグラウンド／worktree 系バグを修正。
- 何が便利か: 長時間のバックグラウンド／リモート実行が途中で認証切れ・アイドル回収されにくくなり、許可モードの取り違えも防げる。MCP サーバーがマルチディレクトリ構成のワークスペースを正しく認識できる。
- 使いどころ: 複数リポジトリ／worktree をまたぐ大規模リファクタや、スマホから監督するバックグラウンドエージェント運用。CI や headless 実行での安定運用。
- 開発者にとっての価値: 「エージェントを回しっぱなしにする」運用の信頼性が上がり、MCP ツールへ渡すファイルスコープが正確になる。
- 注意点: 2.1.204 は headless セッションの SessionStart フックのストリーミング修正が主。恩恵を受けるには CLI を最新へ更新する必要がある。
- 公式リンク: https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md
- 参考リンク: https://code.claude.com/docs/en/changelog
- メモ: 前日（07-08）の AI ニュースレターで扱った Claude Cowork のモバイル／Web 展開とは別トピック（こちらは CLI 本体のリリース）。

#### 使用例

```bash
# 最新へ更新して現在の許可モード／状態を確認
npm install -g @anthropic-ai/claude-code
claude --version
claude /status
```

### 2. GitHub Copilot デスクトップアプリ / 全プラン開放

- 種別: デスクトップアプリ / Webサービス
- 関連: GitHub Copilot
- 公開・更新日: 2026-07-07
- 概要: エージェント駆動開発向けの GitHub Copilot デスクトップアプリが全 Copilot プランで利用可能に。Copilot Free や GitHub Education を含み、macOS / Windows / Linux に対応。Copilot サブスクリプションが無くても BYOK（自分のモデルプロバイダのキー）でセッションを実行できる。
- 何が便利か: GitHub アカウントでサインインするだけで、デスクトップから並行エージェント開発を開始できる。無料プランや自前キーでも試せるため導入ハードルが低い。
- 使いどころ: ローカルのデスクトップ環境でエージェントにタスクを任せたい個人開発者や学生。まず無料枠や BYOK で挙動を確認したいチーム。
- 開発者にとっての価値: ブラウザや IDE 拡張に依存せず、専用デスクトップアプリでエージェント開発を回せる選択肢が全ユーザーに開いた。
- 注意点: Copilot Business / Enterprise で使う場合は、組織／Enterprise 管理者がポリシーで Copilot CLI を有効化している必要がある。
- 公式リンク: https://github.blog/changelog/2026-07-07-github-copilot-app-available-to-all/
- 参考リンク: https://github.blog/news-insights/product-news/github-copilot-app-the-agent-native-desktop-experience/
- メモ: 6/17 の一般提供（GA）に続く「全プラン開放」の続報。GA 時点では対象が限定的だった。

### 3. Antigravity CLI / v1.1.0

- 種別: CLI
- 関連: Google AI / Antigravity（旧 Gemini CLI 後継）
- 公開・更新日: 2026-07-08
- 概要: エージェント実行モードの循環切替（default → accept-edits → plan）が正式公開され、`shift+tab` で切替可能に。起動時にモードを指定する `--mode` フラグを追加。`request-review` がデフォルトモードになり、ファイル書き込み前に行単位の差分プレビューを表示して個別に採否できる。`/plan` が旧 `/planning` を置き換え、`/fast` は削除。`/settings` にデフォルト実行モードを永続化する Agent Mode を追加。
- 何が便利か: エージェントが勝手にファイルを書き換える前に、行単位で差分を確認・採否できるのが標準挙動に。モード切替がキーボードで即座に行える。
- 使いどころ: 大規模リファクタや自動修正で「まず計画→レビュー→適用」を徹底したい場面。安全側に倒したい本番リポジトリでのエージェント運用。
- 開発者にとっての価値: レビュー導線が標準化され、エージェントの変更を人間が制御しやすい。Gemini CLI からの移行組にとって実運用に耐える機能が揃ってきた。
- 注意点: 旧 Gemini CLI は 2026-06-18 に停止済み。`/planning` `/fast` など旧コマンドは使えないため、スクリプトやエイリアスの更新が必要。
- 公式リンク: https://github.com/google-antigravity/antigravity-cli/blob/main/CHANGELOG.md
- 参考リンク: https://developers.googleblog.com/an-important-update-transitioning-gemini-cli-to-antigravity-cli/
- メモ: Go 実装でバックグラウンドの複数エージェント並行実行に対応。個人 Google アカウント（OAuth）で無料枠が使える点は Gemini CLI から継続。

#### 使用例

```bash
# レビュー付きで起動（書き込み前に行単位差分を確認）
agy --mode request-review
# セッション中は shift+tab で default → accept-edits → plan を循環
```
