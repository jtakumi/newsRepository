### 1. OpenAI Codex / v0.145.0（安定版）

- 種別: CLI
- 関連: OpenAI / Codex / Cursor / Claude Code / Amazon Bedrock
- 公開・更新日: 2026-07-21（18:21 UTC / JST 07-22 03:21）※安定版
- 概要: OpenAI のコーディングエージェント CLI の安定版。07-17〜07-21 に v0.145.0-alpha を積み上げてきた系列がついに安定版として公開された。目玉は **`/import` の拡張**で、**Cursor と Claude Code の設定・MCP サーバー・プラグイン・セッション・コマンド・プロジェクト単位のメモリを移行**でき、**プラグインコマンドはインストール時に skill へ変換**される。加えて **Amazon Bedrock ログイン（experimental）**（マネージド Bedrock ログイン API、カスタムエンドポイント/認証、既定モデルは GPT-5.6 Sol）、**音声入力・ツール出力（ローカルの一般的な音声形式）と streaming realtime V3 会話**、**マルチエージェント V2 の安定化**（サブエージェントの**モデル・推論レベル・並列度を設定可能**、ロールを設定したときだけエージェントタイプを spawn、親所有のサブエージェントスレッドは TUI で読み取り専用）、**スレッド履歴のページング（experimental）** を投入。
- 何が便利か: **他ツールからの移行が一発でできる**のがまず効く。Cursor や Claude Code で作り込んだ MCP・コマンド・メモリを手で移し替えずに Codex 側へ持ち込めるため、乗り換え・並行運用のコストが大きく下がる。マルチエージェント V2 が安定版になったことで、**サブエージェントごとにモデルと推論レベル・並列度を割り当てる**構成を本番前提で組める。Bedrock ログインで AWS 課金・権限管理配下に載せやすくなる。
- 使いどころ: Cursor / Claude Code から Codex へ移行・併用したいチーム（`/import` で設定を引き継ぐ）。役割ごとに軽量モデルと高推論モデルを混ぜたマルチエージェント実行。AWS を主軸にしていて Bedrock 経由で GPT-5.6 系を使いたい構成。
- 開発者にとっての価値: 07-21 記事では Codex は **alpha のみ**で見送っていたが、今回**安定版 v0.145.0** が出て採用可能に。移行支援・マルチエージェント・Bedrock という「他エコシステムからの受け皿」を安定版でまとめて入れてきた点が実務的。
- 注意点: **Bedrock ログインと履歴ページングは experimental**、音声は realtime V3 系で環境依存があり得る。マルチエージェント V2 は**ロールを設定して初めてエージェントタイプを spawn** する挙動なので、既存の使い方から設定の追加が要る場合がある。

### 2. Claude Code / v2.1.217（+ v2.1.216）

- 種別: CLI
- 関連: Claude / Anthropic / Claude Code
- 公開・更新日: 2026-07-21（21:35 UTC / JST 07-22 06:35）※ v2.1.216 は 07-20 22:14 UTC / JST 07-21 07:14
- 概要: Anthropic のコーディングエージェント CLI の安定版が連続更新。**v2.1.217** の要点は**サブエージェントの暴走抑止**で、**同時実行数に上限（既定 20、`CLAUDE_CODE_MAX_CONCURRENT_SUBAGENTS` で変更）** を設け、**サブエージェントが入れ子でサブエージェントを spawn する挙動を既定で禁止**（`CLAUDE_CODE_MAX_SUBAGENT_SPAWN_DEPTH` で深さを許可）。あわせて **`--max-budget-usd` がバックグラウンドのサブエージェントも停止する**よう修正（上限到達で新規 spawn を拒否し、実行中の背景エージェントも停止）。運用面では**打ち切った MCP ツール出力が全文をメモリに保持し続けるメモリリーク**、**企業内 mTLS / TLS 検証 / OAuth スコープ / プロキシ設定が Claude Desktop セッションで無視される問題**、**Bedrock 上の Claude Opus 4.8 で auto-compact が発動しない問題**を是正。ほかに絵文字ショートコード補完（`:heart:`→❤️）、トランスクリプト書き込み失敗時の警告なども追加。**v2.1.216** は**ファイルシステム隔離だけを外しネットワーク制御は残す `sandbox.filesystem.disabled` 設定**、**長時間セッションでメッセージ正規化コストが二次関数的に増える遅延の修正**、**worktree 隔離のサブエージェントが git を共有チェックアウトへ誤って向ける問題の修正**を投入。
- 何が便利か: **サブエージェントの同時実行と入れ子生成に明確な上限が付いた**ことで、一つの指示から背景エージェントが際限なく増える事故を防げる。**`--max-budget-usd` が背景ジョブにも効く**ようになり、コスト上限が実際の停止に結び付く。**MCP 出力のメモリリーク解消**で長時間セッションが軽くなり、**企業ネットワーク（mTLS/プロキシ/OAuth スコープ）設定が尊重される**ため、社内環境での接続トラブルが減る。
- 使いどころ: マルチエージェント/背景実行を多用し、同時実行数とコストを制御したい構成。企業プロキシ・mTLS 配下で Claude Desktop を使うチーム。長時間セッションや大きな MCP 出力を扱う運用。
- 開発者にとっての価値: 07-21 記事の v2.1.215（`/verify`・`/code-review` の自動実行停止）に続き、**エージェントの自律実行に上限・予算・隔離のガードを足す**方向の連続更新。暴走とコストの予測可能性がさらに上がる。
- 注意点: サブエージェントの**入れ子生成が既定でオフ**になったため、多段のサブエージェント構成に依存していた場合は `CLAUDE_CODE_MAX_SUBAGENT_SPAWN_DEPTH` の設定が要る。`sandbox.filesystem.disabled` は**隔離を意図的に外す**設定なので、利用時は影響範囲を理解した上で。中心は新機能というより**制御・修正**。
- 公式リンク: https://github.com/anthropics/claude-code/releases/tag/v2.1.217
- 参考リンク: https://github.com/anthropics/claude-code/releases , https://github.com/anthropics/claude-code/releases/tag/v2.1.216
- メモ: 07-22 時点の最新安定版は v2.1.217。`claude update` で反映できる。ログイン期限の警告は「5 日前→3 日前」に変更されている点も細かいが挙動変化。

#### 使用例

```bash
# 更新して最新版を確認
claude update
claude --version            # 2.1.217 を確認

# サブエージェントの同時実行上限を明示（既定は 20）
export CLAUDE_CODE_MAX_CONCURRENT_SUBAGENTS=8

# 予算上限を付けて実行（背景サブエージェントも上限で停止する）
claude --max-budget-usd 2.00
```

### 3. Vercel AI SDK / ai@7.0.34・7.0.33（+ 6.0.233・6.0.232）

- 種別: OSS / SDK（TypeScript）
- 関連: Vercel AI SDK
- 公開・更新日: 2026-07-21（`ai@7.0.34` 15:29 UTC / JST 07-22 00:29、`ai@7.0.33` 12:09 UTC / JST 21:09、`6.0.233 / 6.0.232` も同日）
- 概要: TypeScript 向けの AI アプリ開発 SDK が v7 系・v6 系ともに更新。中身はバグ修正が中心で、**文字起こし（transcription）で MP4 音声を ftyp ボックスから正しく検出**、**同一の tool call ID がステップをまたいで再利用されても tool パーツを保持**、**連続する tool メッセージを結合する際に providerOptions を維持**する修正が入った。`ai@7.0.34` は `@ai-sdk/gateway@4.0.26` への依存更新。
- 何が便利か: **マルチステップのツール実行**で、同じ tool call ID が再登場したときに**ツールの出力パーツが取りこぼされない**ため、エージェント的な繰り返し呼び出しの結果が壊れにくくなる。**providerOptions がメッセージ結合後も維持**されるので、プロバイダー固有設定を付けた会話が意図通り送られる。**MP4 音声の検出修正**で、音声文字起こしのフォーマット判定が安定する。
- 使いどころ: Vercel AI SDK でツール呼び出しを多用するエージェント/チャットアプリ。音声文字起こしを組み込んだ機能。v7 移行済み・v6 継続のどちらの系列でも取り込める。
- 開発者にとっての価値: 07-21 記事では「07-17 以降 24〜48h の新規リリースなし」として見送っていたが、今回**tool 実行・providerOptions・音声検出**という実務で踏みやすい不具合の修正が出たため採用。派手さはないが、ツール実行が絡むアプリでは効いてくる。
- 注意点: 新機能ではなく**バグ修正主体**。v7 系（`ai@7.0.34`）と v6 系（`ai@6.0.233`）で番号が分かれているので、使っている系列に合わせて更新する。
- 公式リンク: https://github.com/vercel/ai/releases
- 参考リンク: https://github.com/vercel/ai/releases/tag/ai%407.0.34 , https://ai-sdk.dev/
- メモ: v6 系にも同等の修正（`6.0.233 / 6.0.232`）がバックポートされている。ツール呼び出しの取りこぼしに心当たりがある場合は優先的に取り込みたい。
