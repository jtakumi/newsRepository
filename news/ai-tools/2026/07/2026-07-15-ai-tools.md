### 1. Claude Code / v2.1.208・v2.1.209

- 種別: CLI
- 関連: Claude / Anthropic / Claude Code
- 公開・更新日: 2026-07-14（v2.1.208 は 01:10 UTC、v2.1.209 は 06:36 UTC）
- 概要: Anthropic のコーディングエージェント CLI が 2 連続でリリース。**v2.1.208** は (1) プレーンテキスト描画の**スクリーンリーダーモード**（`claude --ax-screen-reader`、環境変数 `CLAUDE_AX_SCREEN_READER=1`、または設定 `"axScreenReader": true` でオプトイン）、(2) Vim モードで**2 キー列をリマップする `vimInsertModeRemaps`**（例: `jj` を Esc に割り当て）、(3) 企業のカスタムランチャー対応の **`CLAUDE_CODE_PROCESS_WRAPPER`**、(4) フルスクリーン時のマルチセレクトや「Other」入力行の**マウスクリック対応**を追加。バグ修正として、モデル切替後に fast モードが無効のままになる不具合、バックグラウンドエージェントへの返信欠落、**自動更新中のコンテキストウィンドウ初期化**、supervised/background セッションでの **HTTP/2 接続クラッシュ**、大きな応答での **`stream-json` 出力の切り捨て**、**200 行超の Markdown テーブル描画停止（ページング化）**、Read/Grep/Glob/Edit まわりの複数不具合、Bedrock ストリーミングエラー、長時間セッションのメモリリークなどを解消。**v2.1.209** は `claude agents` のバックグラウンドセッションで `/model` などのダイアログが開けなくなっていた過剰なガードを差し戻して修正。
- 何が便利か: `jj` → Esc のような**手癖のリマップ**が入り、Vim 使いの入力ストレスが減る。スクリーンリーダーモードで支援技術ユーザーの利用が現実的に。CI や `claude agents` 常用者にとっては、**バックグラウンド/supervised 実行の安定化**（HTTP/2 クラッシュ、コンテキスト初期化、ダイアログブロック回帰の解消）が地味に効く。
- 使いどころ: Vim キーバインドで Claude Code を常用する開発者、支援技術を使う開発者、企業ランチャー経由で配布・起動する環境、`claude agents` でバックグラウンドタスクを回す運用。
- 開発者にとっての価値: 前バージョン系（v2.1.206/207）の Auto モード拡充・端末フリーズ修正に続く**入力体験とバックグラウンド実行の安定化**。日常的に回すほど恩恵が大きい修正が中心。
- 注意点: 目玉は新機能というより**アクセシビリティ・入力・安定性**の改善。スクリーンリーダーモードと `vimInsertModeRemaps` はオプトイン設定。`CLAUDE_CODE_PROCESS_WRAPPER` は企業ランチャー連携向けで一般ユーザーには不要なことが多い。
- 公式リンク: https://github.com/anthropics/claude-code/releases/tag/v2.1.209
- 参考リンク: https://github.com/anthropics/claude-code/releases/tag/v2.1.208 , https://github.com/anthropics/claude-code/releases
- メモ: バックグラウンド/デーモン実行まわりの修正が続いており、`claude agents` を実運用に載せる場合は最新版への追随を優先すると安定する。

### 2. GitHub Copilot / `/security-review`（公開プレビュー）

- 種別: 機能追加（GitHub Copilot アプリ内スラッシュコマンド）
- 関連: GitHub Copilot
- 公開・更新日: 2026-07-14
- 概要: GitHub Copilot アプリに **`/security-review` スラッシュコマンド**が公開プレビューで追加。現在の作業差分（workstream changes）を AI が解析し、**重大度と確信度でスコア付けした高確信度のセキュリティ指摘**と、適用可能な修正提案を返す。Copilot から離れずに提案を**適用 → 再検証（reverify）**でき、コードがマージされる前に重要な問題を優先度順に潰せる。スキャンは**インジェクション、クロスサイトスクリプティング（XSS）、安全でないデータ処理、パストラバーサル、弱い暗号**といった影響の大きい脆弱性クラスに調整されている。**Copilot Free / Pro / Business / Enterprise** の全ユーザーがプレビュー期間中に利用可能。
- 何が便利か: 専用のセキュリティスキャナーを別途立てなくても、**日常のコーディングフローの中で差分ベースの脆弱性チェック**を回せる。指摘が重大度・確信度でスコア化されるため、ノイズに埋もれず「まず直すべき箇所」から着手できる。
- 使いどころ: プルリクエスト作成前の自己レビュー、レビュワーの一次スクリーニング補助、CI に本格的な SAST を組む前の軽量な前段チェック、個人開発での「マージ前の一手間」。
- 開発者にとっての価値: セキュリティレビューの入り口が**エディタ内の 1 コマンド**に降りてくる。Free でも使えるため、個人・小規模チームでも脆弱性の作り込みを早期に発見しやすい。
- 注意点: 公開プレビュー段階のため精度・対象範囲は発展途上。あくまで**高確信度の指摘に絞った補助**で、網羅的な監査や専門のセキュリティレビューを置き換えるものではない。対象は「common, high-impact」なクラスに調整されており、ロジック上の欠陥や設計レベルの問題は範囲外になりうる。
- 公式リンク: https://github.blog/changelog/2026-07-14-security-reviews-now-available-in-the-github-copilot-app/
- 参考リンク: https://github.blog/changelog/label/copilot/
- メモ: 同日の GitHub Copilot CLI は 1.0.71-1（プレリリース、MCP ツールセット設定の永続化やプラグインマーケットプレイス subcommand を追加）。安定版は v1.0.70（07-10）のため、CLI 側は今回は本文に採用せず参考にとどめる。

### 3. Vercel AI SDK / `ai@7.0.27`・`7.0.28`

- 種別: OSS / SDK
- 関連: Vercel AI SDK
- 公開・更新日: 2026-07-14（`ai@7.0.28` は 21:35、`ai@7.0.27` は 18:21、バックポート `6.0.226` / `5.0.213` は 18:18〜18:19）
- 概要: `ai@7.0.28` は、**リクエストが重なって再開（resume）ストリームの完了前にアクティブ応答がクリアされたときのチャット `onFinish` の扱い**を修正。`ai@7.0.27` は、(1) 永続化されたエラー応答向けに**空パーツを含むアシスタント UI メッセージの検証**を許可、(2) `experimental_streamTranscribe` の結果 Promise が **`fullStream` を消費せずに解決できるよう修正**、あわせて `@ai-sdk/provider-utils` / `@ai-sdk/gateway` を更新。6 系（`6.0.226`）・5 系（`5.0.213`）にも、**モデルストリームエラー後に保留中ツール実行が結果を積んでしまう不具合**の修正と同等の UI メッセージ検証改善をバックポート。
- 何が便利か: **並行リクエスト時のコールバック取りこぼし**や、ストリーミング文字起こしで `fullStream` を読まないと結果が返らない、といった実装でハマりやすい挙動が解消。エラー応答を永続化して再表示する UI の堅牢性も上がる。
- 使いどころ: 応答の再開・並行送信を扱うチャット UI、`experimental_streamTranscribe` を使う音声文字起こし、ツール呼び出しを含むエージェントでストリームエラーの後始末が必要な構成。
- 開発者にとっての価値: 07-13 分（MCP ツールドリフト検知・`repairToolCall` 昇格）、07-14 前半（トレーシング・文字起こしキャンセル）に続く**エラーハンドリングと UI 状態整合の継続改善**。5/6/7 の各系列に修正が届くため、旧メジャーに留まっている構成でも取り込める。
- 注意点: 今回分は主に**バグ修正・堅牢化のインクリメンタルな更新**で、大きな新 API ではない。`experimental_` プレフィックスの API は今後変わりうる。
- 公式リンク: https://github.com/vercel/ai/releases
- 参考リンク: https://github.com/vercel/ai/blob/main/packages/ai/CHANGELOG.md , https://ai-sdk.dev/
- メモ: 「モデルストリームエラー後に保留中ツール実行が結果を積む」問題への修正が 5/6/7 系すべてに入っており、ツール実行を伴うエージェントを本番運用している場合はメジャーを問わず追随する価値がある。
