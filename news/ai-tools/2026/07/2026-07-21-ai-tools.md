### 1. Claude Code / v2.1.215

- 種別: CLI
- 関連: Claude / Anthropic / Claude Code
- 公開・更新日: 2026-07-19（02:56 UTC / JST 11:56）
- 概要: Anthropic のコーディングエージェント CLI の安定版。今回の変更点は明確で、**Claude が `/verify` スキルと `/code-review` スキルを自分の判断で自動実行しなくなり、ユーザーが明示的に呼び出したときだけ走る**ようになった。直前の v2.1.212〜v2.1.214 が権限ハードニングと長時間実行制御を大量に入れた回だったのに対し、今回は**エージェントの自発的なスキル起動を絞る**方向の挙動調整。
- 何が便利か: これまでは編集後などに Claude が自発的に検証・コードレビューのスキルを走らせ、その分の待ち時間・トークン・ツール実行が発生することがあった。今後は**必要なときだけ手動で呼ぶ**ので、実行タイミングとコストを開発者側で制御でき、意図しない自動レビューで作業が横道にそれるのを防げる。
- 使いどころ: CI や非対話（print / SDK）で Claude Code を回していて、実行コスト・所要時間を予測可能にしたい構成。検証やレビューを自分のフローの特定ポイントでだけ実行したいチーム。
- 開発者にとっての価値: 自動実行を任せる範囲を絞る変更で、**「勝手に動くエージェント」から「呼んだら動くエージェント」へ**の微調整。トークン/実行時間の見積もり精度が上がる。
- 注意点: これは**新機能ではなく挙動変更**。これまで自動レビュー/自動検証に依存していたワークフローがあると、**更新後は明示的に `/verify` や `/code-review` を呼ばないと走らなくなる**ので、フローに組み込んでいた場合は手動呼び出しを追加する必要がある。
- 公式リンク: https://github.com/anthropics/claude-code/releases/tag/v2.1.215
- 参考リンク: https://github.com/anthropics/claude-code/releases , https://releasebot.io/updates/anthropic
- メモ: 07-21 時点の最新安定版は v2.1.215。Anthropic の公開リリースノート（07-19 更新）でも「Claude Code は `/verify` と `/code-review` を直接呼び出したときのみ実行し、Claude 自身では走らせない」と明記されている。

#### 使用例

```bash
# 更新して最新版を確認
claude update
claude --version          # 2.1.215 を確認

# 今後は検証・レビューを明示的に呼び出す（自動では走らない）
/verify
/code-review
```

### 2. Cline / CLI v3.0.45・v3.0.46・SDK v0.0.65・VS Code 拡張 v4.0.10

- 種別: CLI / SDK / 拡張機能（OSS）
- 関連: Cline
- 公開・更新日: 2026-07-19（SDK v0.0.65 は 04:02 UTC、CLI v3.0.45 は 04:10 UTC、CLI v3.0.46 は 06:26 UTC）／ 2026-07-20（VS Code 拡張 v4.0.10 は 07:58 UTC）
- 概要: Cline の CLI / SDK / 拡張が連続更新。目玉は **CLI v3.0.45** の**インストールサイズ削減**で、`npm i -g cline` のダウンロードが**約 640MB → 約 285MB**に縮小（Claude Code / Codex のプロバイダーを**オプションの依存**にしてオンデマンド読み込みに変更）。あわせて **Kimi K3 を ClinePass モデルに追加**、**OAuth 資格情報が失効したら一度リフレッシュしてから実行を再試行**、**Hub のステータス出力にバージョン番号を表示**。**CLI v3.0.46** は**クレジット切れ（Cline API の `insufficient_credits` / 402）を確実に検出**し、汎用エラーではなく「クレジット追加」カードを出すよう修正。**SDK v0.0.65** は Claude Code / Codex プロバイダー SDK を**オンデマンドの optional 依存**にして SDK 側のインストールも軽量化し、**Kimi K3 追加**・**OAuth リフレッシュ後の再試行**・**チーム実行での spawn ツール制限とエラー時の failed 報告**・**Windows 向けのシェルコマンド解析改善**・**`@cline/ui` エージェントチャットコンポーネント（Storybook / npm 提供）**を投入。
- 何が便利か: **導入が軽くなる**のがまず効く（CI キャッシュ・コンテナイメージ・個人環境のディスク/回線に優しい）。OAuth 失効時の**自動リフレッシュ&再試行**で、長時間ジョブが資格情報切れで落ちにくくなる。402 の検出改善で「なぜ止まったか（＝クレジット不足）」が一目で分かる。Kimi K3 を ClinePass 経由でそのまま試せる。
- 使いどころ: Cline CLI を CI / コンテナで回す構成（イメージサイズと初回インストールが軽くなる）、ClinePass で複数モデルを切り替える運用、Windows でシェルコマンドを多用するエージェント実行。
- 開発者にとっての価値: 07-19 記事の v3.0.43/44・SDK v0.0.64（証明書ストア自動取り込み・BOM 対応）に続く**運用摩擦・フットプリントの低減**が主眼。インストールサイズ半減は地味だが、イメージビルドや大量ワーカー環境で効いてくる。
- 注意点: プロバイダーが**オンデマンド読み込み**になったため、Claude Code / Codex プロバイダーを使う場合は**初回に追加取得**が走る（オフライン前提の環境では事前取得を検討）。中心は新機能というより**軽量化・信頼性改善**。
- 公式リンク: https://github.com/cline/cline/releases
- 参考リンク: https://github.com/cline/cline , https://github.com/cline/cline/releases/tag/cli-v3.0.45
- メモ: 07-21 時点で CLI は v3.0.46、SDK は v0.0.65、VS Code 拡張は v4.0.10 が最新。既存環境なら `npm i -g cline@latest` で軽量版に入れ替えできる。

#### 使用例

### 3. LiteLLM / v1.93.0（安定版）

- 種別: OSS / API ゲートウェイ（プロキシ）
- 関連: LiteLLM / MCP / Anthropic / GPT-5.6
- 公開・更新日: 2026-07-19（07:57 UTC / JST 16:57）※安定版
- 概要: 100+ の LLM を OpenAI 互換 API に集約するプロキシ/SDK の安定版。**MCP 周りの認証を強化**し、**OAuth2 トークンエクスチェンジ（entra_obo プロファイル）**と **dcr_bridge の認証モード（`true_passthrough` / `oauth_delegate`）**、MCP のセマンティックフィルタリング改善を追加。ルーターは**デプロイ単位で ITPM/OTPM（入力/出力トークン毎分）を分けて制限**でき、**個別キーへのタグ別 RPM 制限**にも対応。セキュリティ面では**スペンドログ内の `Bearer ` 付き API キーをハッシュ化**。修正として**ストリーミング `/v1/messages` 応答でプロバイダーの応答ヘッダーを転送**、**Bedrock ストリームが途中終了したときの SSE エラーイベント**、**CacheCodec が null フィールドを落とす問題**などを是正。モデル面では **GPT-5.6 バリアント**や Vertex AI の Chirp 3 音声認識サポートを追加。
- 何が便利か: **MCP をエンタープライズ認証（Entra / OAuth2）配下で使う構成**で、トークンエクスチェンジや委譲モードが素直に扱えるようになる。**ITPM/OTPM をデプロイ別に分離**できるため、入力偏重・出力偏重のワークロードを同じルーターで別々に絞れる。スペンドログの**キー・ハッシュ化**でログ経由の資格情報漏えいリスクが下がる。ストリーミングのヘッダー転送で、上流プロバイダーのレート情報などを下流に渡せる。
- 使いどころ: 社内で複数モデルを LiteLLM に集約し、チーム/キー単位でレート・コストを管理している構成。MCP サーバーを OAuth2/Entra 認証込みでぶら下げるエージェント基盤。GPT-5.6 系を LiteLLM 経由で呼ぶ環境。
- 開発者にとっての価値: 07-19 記事時点では LiteLLM は**プレリリースのみ**で見送りだったが、今回**安定版 v1.93.0** が出たため採用可能に。認証・レート制御・ログのセキュリティという**運用の要所**がまとまって入る。
- 注意点: 同日に **v1.92.1 / v1.91.4 / v1.90.6** のバックポート安定版も出ている（既存系列に留まる場合の取り込み用）。**v1.94.0-rc.1 は RC（プレリリース）**で、Anthropic の自動プロンプトキャッシュ（Admin UI 設定）や複雑度ルーター、予算強制などを含むが本番採用は安定版の v1.93.0 が無難。
- 公式リンク: https://github.com/BerriAI/litellm/releases/tag/v1.93.0
- 参考リンク: https://github.com/BerriAI/litellm/releases , https://docs.litellm.ai/
- メモ: MCP を認証付きで運用しているなら OAuth2 トークンエクスチェンジ回りが目玉。スペンドログのキー・ハッシュ化はセキュリティ改善なので、ログを外部に流している構成では優先的に取り込みたい。
