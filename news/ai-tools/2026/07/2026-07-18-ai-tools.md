### 1. Claude Code / v2.1.212

- 種別: CLI
- 関連: Claude / Anthropic / Claude Code
- 公開・更新日: 2026-07-17（00:26 UTC）
- 概要: Anthropic のコーディングエージェント CLI の新版。目玉は会話・並列実行まわりの新コマンド。**`/fork`** は現在の会話を**新しいバックグラウンドセッションへコピー**しつつ現行セッションをそのまま継続でき、**`/subtask`** は**セッション内でサブエージェント**を起動する。運用系では **`claude auto-mode reset`**（確認プロンプト付き、`--yes` でスキップ）、**セッション単位の WebSearch 上限**（既定 200、`CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION`）、**サブエージェント生成数の上限**（既定 200、`CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION`）、**2 分超の MCP ツール呼び出しの自動バックグラウンド化**（`CLAUDE_CODE_MCP_AUTO_BACKGROUND_MS`）を追加。`/resume` は**削除済みを含む過去セッションのピッカー**に。バグ修正として、**plan モードがファイル変更 Bash を権限確認なしに自動実行**してしまう問題、print/SDK モードで **SIGTERM がプロセスツリーを孤児化**する問題（終了コード 143 で正しく終了）、`/ultrareview` が `#123` や `PR 123` 形式の PR 参照を拒否する問題を解消。加えて **LLM ゲートウェイ／カスタム base URL 経由でのプロンプトキャッシュ改善**、`SendMessage` 本文の重複排除によるエージェント間通信のトークン削減も。
- 何が便利か: `/fork` で「今の会話を壊さずに別方向を試す」分岐がワンコマンドででき、`/subtask` で重い調査を並列に切り出せる。長時間の MCP ツールが自動でバックグラウンド化されるため、対話がブロックされにくい。plan モードの権限バイパス修正は、意図しないファイル変更を防ぐ安全面の改善。
- 使いどころ: 会話を分岐させて複数案を並行検討したいエージェント運用、MCP ツールを多用する構成、CI／ゲートウェイ経由でのキャッシュ最適化、plan モードでレビュー主体に回す使い方。
- 開発者にとっての価値: 07-14 系（アクセシビリティ・入力・安定性）に続き、**並列・分岐ワークフローとセッション制御**へ踏み込んだ更新。エージェントを実運用で回すほど恩恵が大きい。
- 注意点: `/fork`・`/subtask` は新ワークフロー前提のため使い方に慣れが必要。WebSearch／サブエージェント上限やキャッシュ挙動は環境変数で調整可能だが、デフォルト値の把握が前提。
- 公式リンク: https://github.com/anthropics/claude-code/releases/tag/v2.1.212
- 参考リンク: https://github.com/anthropics/claude-code/releases
- メモ: plan モードのファイル変更バイパス修正は安全面で重要。plan モードを「読み取り主体」で運用しているチームは早めに追随したい。

### 2. GitHub Copilot CLI / v1.0.71（安定版）

- 種別: CLI
- 関連: GitHub Copilot
- 公開・更新日: 2026-07-16（01:54 UTC）
- 概要: GitHub Copilot のコマンドライン版が安定版 v1.0.71 に到達。安全面の目玉として、**plan モード中は組み込みツールによるワークスペース変更呼び出しをハードブロック**し、計画中にエージェントがファイル編集や破壊的シェルを実行できないようにした。安定性では、**バックグラウンドプロセスが長引くと autopilot モードがハングする不具合**を修正。機能面では **`/worktree`・`/move` によるワークスペースディレクトリ管理**、**プラグインのマーケットプレイス閲覧・更新**、**GitHub MCP ツールセット／ツール設定を `settings.json` に永続化**、ボイスモードのマイクデバイス選択の保持、サイドバーセッションの再起動をまたいだ永続化などを追加。`settings.json` が不正な場合に**該当値を指摘する警告**を出すようになり、`/terminal-setup` の端末互換性や選択メニューでの vim キー挙動も改善。
- 何が便利か: plan モードのハードブロックにより「計画のつもりが実際に書き換わっていた」事故を構造的に防げる。長時間バックグラウンドジョブでの autopilot ハングが解消し、CLI 常用時の安定性が上がる。MCP ツールセット設定を `settings.json` で永続化できるため、環境を再現しやすい。
- 使いどころ: ターミナル主体で Copilot を回す開発者、plan → 実行を分けて安全にレビューしたい運用、複数ワークツリーを跨ぐ作業、MCP を組み込んだ再現性重視の環境構築。

- 注意点: `v1.0.72-0/-1`（07-16〜17）は**プレリリース**（マルチターンサブエージェント等）で本文の対象外。安定運用は v1.0.71 を基準に。

- 参考リンク: https://github.com/github/copilot-cli/blob/main/changelog.md , https://github.blog/changelog/label/copilot/
- メモ: plan モードのワークスペース保護は Claude Code v2.1.212 の plan モード修正と同方向の動き。エージェント CLI 全般で「計画中は書き換えさせない」設計が標準化しつつある。
