# 2026-07-11 AI Tools News

## 今日の要点

- **主要エージェント CLI が同日更新**。**Claude Code v2.1.206**（07-09）と **OpenAI Codex CLI v0.144.0 / v0.144.1**（07-09）が相次いで公開。どちらも「PR 連携・認証・権限モード」の実運用まわりを強化しており、既存ワークフローの安全性と自動化度が今日から上がる。
- Claude Code は **`/commit-push-pr` が設定済み push リモートへの `git push` を自動許可**、MCP サーバーの**サーバー別 `request_timeout_ms`** 対応、**OAuth MCP のリフレッシュ失敗後の再認証不要化**など、無人・長時間運用に効く改善が中心。

- ローカル LLM 側は **Ollama v0.31.2** が**古い NVIDIA GPU（compute capability 6.x）での flash attention**、**iGPU でのビジョンモデル offload**に対応。手元の旧世代・内蔵 GPU でもマルチモーダル推論を回しやすくなった。
- Google の **Gemini CLI v0.50.0**（07-08）は **Tool Registry Discovery**（ツールの自動検出・登録）を導入。自作ツールや MCP を CLI へ組み込む拡張性が向上した。

## 今日の注目

### 1. Claude Code / v2.1.206

- 種別: CLI
- 関連: Anthropic / Claude Code
- 公開・更新日: 2026-07-09（v2.1.206）
- 概要: エージェント運用まわりを広く強化するメンテナンスリリース。**`/commit-push-pr` が設定済みの push リモートへの `git push` を自動許可**し、コミット→push→PR 作成を一気通貫でよりスムーズに。**MCP サーバーがサーバー別の `request_timeout_ms` 設定を尊重**するようになり、**OAuth 系 MCP サーバーはリフレッシュ失敗後の手動再認証が不要**に。`/cd` にディレクトリ候補補完（`/add-dir` と同挙動）、`/doctor` にチェックイン済み `CLAUDE.md` の肥大を指摘するチェックを追加。`EnterWorktree` はプロジェクト外の worktree に入る前に確認を挟むようになり、**バックグラウンドエージェントは Claude Code 更新後に自動アップグレード**。`/model` ピッカーの価格表示ずれ、Windows でのキーボード入力無視、Bedrock 起動時のハングなど多数の修正。**`claude-opus-4-8` 上での `/code-review` 品質も改善**。
- 何が便利か: PR まわりの権限確認が減り、`/commit-push-pr` の自動化がより実用的に。MCP のタイムアウトを個別調整でき、OAuth 連携が落ちても再ログインを求められにくいので、無人・長時間セッションが安定する。
- 使いどころ: headless／バックグラウンドでの PR 自動作成パイプライン、複数 MCP サーバー（応答速度がばらつく外部 API 連携）の運用、`/doctor` によるリポジトリ健全性チェック。
- 開発者にとっての価値: 権限・認証・タイムアウトという「無人運用でつまずく箇所」を一括で潰しており、CI やスケジュール実行に組み込むエージェント作業の信頼性が上がる。
- 注意点: 恩恵を受けるには CLI を最新へ更新する必要がある。前日（07-10）記事は v2.1.205 までを扱ったため、本記事は v2.1.206 が新規分。`/commit-push-pr` の自動 push 許可は対象が「設定済み push リモート」に限られる点を理解して使う。
- 公式リンク: https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md
- 参考リンク: https://code.claude.com/docs/en/changelog
- メモ: MCP のサーバー別タイムアウトと OAuth 再認証不要化は、外部 API 連携を多用するエージェント構成に直接効く。

#### 使用例

```bash
npm install -g @anthropic-ai/claude-code
claude --version          # 2.1.206 を確認
claude /doctor            # CLAUDE.md 肥大チェックを含む総点検
# コミット→push→PR を一気通貫（push は設定済みリモートへ自動許可）
claude /commit-push-pr
```

### 2. OpenAI Codex CLI / v0.144.0・v0.144.1

- 種別: CLI
- 関連: OpenAI / Codex
- 公開・更新日: 2026-07-09（v0.144.0 → v0.144.1）
- 概要: v0.144.0 は**新しいアプリ承認モード**と**対話型認証（interactive auth）**を追加し、**Code Mode とサンドボックスセッションの不具合**を修正（21 機能・17 改善・27 バグ修正など計 96 変更）。同日リリースの v0.144.1 は、**GitHub がコンパクト／並び替えされたリリースメタデータを返す際にスタンドアロンインストールが失敗する不具合**を修正し、macOS のパッケージインストールで code-mode ホストを codex 実行ファイルと並べて配置、コンパニオンホストが無い環境でも埋め込みランタイムへフォールバックして Code Mode を維持する。
- 何が便利か: 承認モードと対話型認証で、権限付与・ログインの取り回しが改善。Code Mode まわりの安定化と、インストール失敗の修正で導入・更新の詰まりが減る。
- 使いどころ: Codex CLI をローカル／CI に導入した環境、Code Mode を使うコード実行タスク、macOS での新規セットアップ。
- 開発者にとっての価値: v0.144.1 のインストール修正で更新の失敗リスクが下がり、v0.144.0 の承認モード追加で権限運用の細かい制御がしやすくなる。
- 注意点: 前日（07-09）記事は v0.143.0（07-08）を扱ったため、本記事は v0.144.x が新規分。承認モードの挙動は運用ポリシーに合わせて確認してから自動化に載せる。
- 公式リンク: https://developers.openai.com/codex/changelog
- 参考リンク: https://github.com/openai/codex/releases
- メモ: 同日に .0 と .1 が出ており、更新するなら v0.144.1 まで上げてからインストール不具合を回避するのが安全。

#### 使用例

```bash
# 最新へ更新（v0.144.1 を確認）
codex --version
# 対話型認証でログイン
codex login
```

### 3. Ollama / v0.31.2

- 種別: OSS / ローカルLLM
- 関連: Ollama / ローカルLLM
- 公開・更新日: 2026-07-08 前後（v0.31.2）
- 概要: **古い NVIDIA GPU（compute capability 6.x）で flash attention を有効化**し、**iGPU（内蔵 GPU）がビジョンモデルを利用可能メモリに収まるようパディングして offload**できるようになった。あわせて thinking モデルの structured output 修正、GGUF モデル作成の堅牢化、`ollama launch` で Claude Code を起動する際に**既定でテレメトリを無効化**する変更など。
- 何が便利か: 旧世代 NVIDIA GPU や内蔵 GPU しかない手元マシンでも、flash attention による省メモリ・高速化とマルチモーダル（ビジョン）推論が回しやすくなる。ハイエンド GPU を持たない開発者の実験環境が広がる。
- 使いどころ: ローカルでのマルチモーダル推論の検証、旧 GPU／ノート PC の内蔵 GPU での軽量ローカル LLM 運用、`ollama launch` からのローカルエージェント起動。
- 開発者にとっての価値: クラウド課金なしで手元 GPU を使い切れる。ビジョンモデルの offload 対応で、画像入力を伴うプロトタイプをローカルで試しやすい。
- 注意点: flash attention の対象は compute capability 6.x 世代の NVIDIA GPU。iGPU の offload は利用可能メモリ次第でパディング動作が入る。同時期（07-09）に発表された $65M シリーズ B の資金調達はツール実用性とは無関係のため本記事では扱わない。日付は v0.31.2（07-06〜08 リリース）で 24〜48h の境界にあるが、Ollama の最新安定版として採用。
- 公式リンク: https://github.com/ollama/ollama/releases
- 参考リンク: https://ollama.com/blog
- メモ: 資金調達（投資ニュース）は除外条件に該当。技術リリースのみを採用対象とした。

#### 使用例

```bash
# 更新して手元 GPU でビジョンモデルを試す
ollama --version          # 0.31.2 を確認
ollama run gemma4:vision "この画像の内容を説明して"
```

### 4. Gemini CLI / v0.50.0

- 種別: CLI / OSS
- 関連: Google AI / Gemini CLI
- 公開・更新日: 2026-07-08（v0.50.0、安定版）
- 概要: 最新安定版 v0.50.0 が **Tool Registry Discovery**（ツールレジストリの自動検出・登録）を導入。ツールを自動で発見・登録して拡張性を高める仕組みで、実装は PR #28113 として取り込まれた。あわせて `npm ci` 時のスクリプト無視やワークスペースバイナリのシャドーイング防止といったリリース検証の改善、不良 NPM リリースを防ぐ CI 保護も追加。
- 何が便利か: 自作ツールや外部ツールを Gemini CLI へ組み込む際、手動登録の手間が減り、拡張がしやすくなる。リリース検証の強化で配布物の信頼性も向上。
- 使いどころ: Gemini CLI にカスタムツール／MCP を接続したエージェント構成、社内ツール群を CLI から一元的に呼び出したい場面。
- 開発者にとっての価値: ツール登録の自動化で、エージェントに渡す道具立ての管理コストが下がる。CLI を土台にした自動化を組みやすくなる。
- 注意点: v0.50.0 は 07-08 リリースで、基準日（JST 07-11 朝）から見ると 24〜48h の境界にあるが、Gemini CLI の最新安定版として採用。v0.51 系は本記事時点で nightly／preview 段階。
- 公式リンク: https://github.com/google-gemini/gemini-cli/releases/tag/v0.50.0
- 参考リンク: https://geminicli.com/docs/changelogs/latest/
- メモ: Tool Registry Discovery は、MCP／カスタムツールを多用する構成の「道具の見つけやすさ」に効く基盤機能。
